- 本番想定の主要導線: トップ、可否チェッカー、物件候補リンク集、エリア検索、届出マップ、届出施設一覧、料金プラン、登録/ログイン、問い合わせ、物件掲載、レポート

## 今回修正した内容

1. SUUMOリンク
   - 旧URL `https://suumo.jp/chintai/?fw=...` はSUUMOのトップ/404系画面に寄るケースがあった。
   - `https://suumo.jp/jj/chintai/ichiran/FR301FC005/?ar=...&bs=040&fw=...` に変更。
   - 都道府県ごとにSUUMOの地域コードを付けるようにした。

2. アットホームリンク
   - 旧URL `https://www.athome.co.jp/chintai/?keyword=...` は空の中間ページへ遷移するケースがあった。
   - 確認済みの主要エリアは市区ページへ直リンク。
   - 未確認エリアは死にリンクを避けるため、都道府県の賃貸トップへフォールバック。

3. 課金導線
   - 未ログインのままStripe Checkoutへ進む流れをやめた。
   - 有料プラン選択時は、先に無料アカウント登録へ誘導。
   - 登録/ログイン/メール認証後に、元の料金プランへ戻れる `next` パラメータを保持。
   - レポート経由で決済した場合の成功URL生成も修正。

4. メール認証完了ページ
   - `/auth/complete` に完了ページを作成済み。
   - `next=/pricing?...` がある場合は「料金プランへ進む」を表示。
   - 認証ハッシュ処理後も戻り先クエリを消さないようにした。

5. セレクトボックス表示
   - 都道府県などの選択肢が白抜きに見える問題に対して、`select` と `option` の文字色/背景色をグローバルに固定。

6. 物件候補の初期方針
   - 「民泊可能の空き物件」をYADOKARIが断定掲載するのではなく、まずは公式届出済み施設一覧/マップから実運用エリアを見せる。
   - 空き物件探しは「この周辺で賃貸物件を探す」と「外部の民泊可・相談可物件リンク集」へつなげる。
   - 物件掲載は審査制の申請導線として残し、確認済み候補として育てる。

7. 届出施設一覧
   - `/listings` を `/map` へのリダイレクトから、公式届出済み施設の一覧/マップ画面へ変更。
   - 画面内に「これは現在空いている民泊可物件ではない」旨を明記。
   - 施設選択後、SUUMO/アットホーム/YADOKARI分析へ進める導線を追加。

8. 外部リンク生成
   - `東京都港区` のように都道府県と市区が結合されたデータを `東京都 + 港区` に正規化。
   - SUUMO検索語の重複を解消。
   - アットホームの港区リンクを都道府県トップではなく `tokyo/minato-city/list/` に向けるよう補正。

## 使ってみての課題

- 未確認のアットホーム市区ページは、無理に推測せず都道府県ページへ逃がしている。完全な市区直リンク化には、アットホーム側のURLスラッグを個別検証する必要がある。
- 料金プランは本番Stripe環境で最終確認が必要。コード導線は修正済みだが、本番決済は実カード/少額購入またはStripe推奨の本番確認手順で見る。
- Search Console/Analyticsは登録済みでも、データ反映には時間がかかる。
- `/api/health` はローカルでは外部連携不足により `degraded` になり得る。本番ではVercel環境変数で判定する。
- 物件掲載マーケットプレイスは審査制で育てる。実物件公開前は、外部リンク集と届出データ中心の状態が安全。
- モック物件データはユーザーに誤認させない形へ寄せる。実データの掲載開始までは、公開面では「候補/外部リンク/審査制」を前面に出す。
